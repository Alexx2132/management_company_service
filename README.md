**Management Company Service** — backend-часть информационной системы управляющей компании для учета проблем в сфере ЖКХ, обработки заявок жителей и координации работы сотрудников.

Проект разработан в рамках выпускной квалификационной работы на тему:

> «Разработка приложения управляющей компании для учета проблем в сфере ЖКХ»

---

## О проекте

Система предназначена для взаимодействия жителей, диспетчеров, исполнителей, аудиторов, администратора и помощника администратора при создании, обработке и контроле заявок в сфере жилищно-коммунального хозяйства.

Backend предоставляет REST API для web-версии и Android-приложения, хранит данные в PostgreSQL, управляет файлами заявок и жалоб, отправляет уведомления и поддерживает обновление данных в реальном времени через WebSocket.

---

## Основные возможности

### Заявки жителей

- создание заявок через web-версию и мобильное приложение;
- создание заявки диспетчером по обращению жителя;
- выбор категории, места возникновения проблемы и приоритета;
- прикрепление фотографий до устранения проблемы;
- назначение исполнителя, принятие заявки в работу и отметка о выполнении;
- фотоотчет исполнителя после устранения проблемы;
- принятие работы жителем, отзыв или жалоба по заявке;
- история статусов с комментариями и действиями участников.

### Жалобы и обращения

- подача жалобы по качеству выполненной работы;
- подача жалобы при отсутствии реакции на заявку;
- обращение в контролирующий орган при неудовлетворительном рассмотрении;
- переписка внутри обращения;
- прикрепление фотографий к жалобе;
- настройка лимитов и временных порогов рассмотрения.

### Пользователи и роли

- администратор управляет пользователями, домами, настройками сервиса и справочными данными;
- помощник администратора получает настраиваемые полномочия;
- диспетчер обрабатывает заявки, назначает исполнителей и рассматривает обращения;
- исполнитель принимает заявки, ведет фотоотчет и видит замечания;
- житель создает заявки, отслеживает статусы, оставляет отзывы и обращения;
- аудитор рассматривает обращения в контролирующий орган.

### Дома, объявления и информация для жителей

- учет домов, подъездов, квартир и жителей;
- адресная публикация объявлений по домам и подъездам;
- архивирование объявлений;
- ведение плановой информации по дому;
- хранение полезных контактов и графиков обслуживания;
- уведомления жителям только по относящимся к ним домам.

### Уведомления и обновления

- push-уведомления в Android-приложении через Firebase Cloud Messaging;
- WebSocket-обновления для web-версии;
- уведомления о новых заявках, жалобах, сообщениях и объявлениях;
- отметки о непрочитанных сообщениях и обращениях.

## Технологическая основа backend

- **FastAPI** — разработка REST API;
- **PostgreSQL** — основная база данных;
- **SQLAlchemy** — работа с моделями и запросами;
- **Alembic** — управление миграциями базы данных;
- **JWT** — авторизация пользователей;
- **Firebase Cloud Messaging** — отправка push-уведомлений;
- **WebSocket** — обновление данных в реальном времени;
- **файловое хранилище** — хранение изображений заявок, жалоб и настроек оформления.

## Структура backend-проекта

```text
uk_backend/
├── app/
│   ├── api/              # REST API и endpoints
│   ├── core/             # конфигурация и безопасность
│   ├── db/               # подключение к базе данных
│   ├── models/           # модели SQLAlchemy
│   ├── repositories/     # слой доступа к данным
│   ├── schemas/          # Pydantic-схемы
│   └── services/         # бизнес-логика приложения
├── alembic/
│   └── versions/         # миграции базы данных
├── static/               # загружаемые файлы и изображения
├── alembic.ini           # настройки Alembic
└── docker-compose.yml    # конфигурация PostgreSQL
```

## Запуск backend

Перед запуском необходимо настроить переменные окружения и подключение к PostgreSQL.

```powershell
python -m alembic upgrade head
python -m uvicorn app.main:app --reload
```

После запуска API доступно по адресу:

```text
http://127.0.0.1:8000
```

Документация FastAPI:

```text
http://127.0.0.1:8000/docs
```
